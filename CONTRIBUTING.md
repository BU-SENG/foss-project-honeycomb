# Contributing

Thanks for your interest in contributing to this project!

This document provides a short guide for making contributions, and lists of contributors that worked on the project.

## Quick Start

- Fork the repository and create a feature branch from `main`:
  - `git checkout -b feat/your-feature`
- Make your changes in the feature branch.
- Run the dev server and tests locally to verify your changes.
- Commit with clear messages and push your branch to your fork.
- Open a Pull Request to `main` and include a concise description of your changes.

## Code style

- The project uses TypeScript, React and Tailwind CSS.
- Keep changes small and focused; prefer readability.
- Follow existing patterns (hooks, context API) and naming conventions.

## Tests

- Add unit or integration tests where appropriate.
- Run existing tests locally before opening a PR.

## Review process

- PRs will be reviewed by maintainers; please address review comments promptly.
- Squash or tidy commits if requested.
